# Development Workflow

How to make changes day-to-day. See also [contributing.md](./contributing.md) (and the root
[CONTRIBUTING.md](../CONTRIBUTING.md)) and [coding-standards.md](./coding-standards.md).

## Branching

Branch from `main`. Use a typed prefix matching the change:

| Prefix | For |
|---|---|
| `feat/…` | new features |
| `fix/…` | bug fixes |
| `chore/…` | tooling, deps, housekeeping |
| `docs/…` | documentation only |
| `refactor/…` | behavior-preserving restructuring |

Example: `feat/gdelt-news-source`.

## Conventional Commits

All commits follow [Conventional Commits](https://www.conventionalcommits.org/):

```text
type(scope): summary
```

- **type**: `feat`, `fix`, `chore`, `docs`, `refactor`, `test`, `perf`, `build`, `ci`.
- **scope**: the area touched (`collectors`, `memory`, `query`, `frontend`, `auth`, …).
- **summary**: imperative mood, lower case, no trailing period.

Examples:

```text
feat(collectors): add GDELT news source
fix(memory): scope dataset by ticker
docs(api): document /companies/{ticker}/query response shape
```

## `make` targets

The [Makefile](../Makefile) is the task runner:

| Target | What it does |
|---|---|
| `make up` | Start the full stack via Docker Compose. |
| `make down` / `make restart` | Stop / restart the stack. |
| `make logs` / `make ps` | Tail logs / list services. |
| `make migrate` | `alembic upgrade head`. |
| `make migration m="…"` | Create an autogenerated migration. |
| `make seed` | Seed demo companies (AAPL, MSFT, TSLA) + a demo user. |
| `make lint` | ruff + black --check + mypy (backend) and eslint + prettier --check (frontend). |
| `make fmt` | Auto-format both apps. |
| `make typecheck` | mypy + tsc. |
| `make worker` / `make beat` | Run a Celery worker / Beat scheduler locally (task bodies are stubs). |
| `make backfill t=AAPL` | Backfill a ticker (stub today). |
| `make purge t=AAPL` | Purge a Cognee dataset (stub today). |
| `make install` | Install backend + frontend deps. |
| `make clean` | Remove caches and build artifacts. |

Run `make help` for the full list. *(There is no `make test` or `make hooks` target yet.)*

## Pull request flow

1. Create a branch from `main`.
2. Make focused, atomic commits (Conventional Commits).
3. Run `make lint` locally — it must pass. (No CI pipeline exists yet — run checks locally.)
4. Push and open a PR; fill in the template (what / why / testing / screenshots).
5. Update [docs](./README.md) and [`CHANGELOG.md`](../CHANGELOG.md) for any user-facing change; new
   env vars go in [`.env.example`](../.env.example) **and** [environment.md](./environment.md); new
   endpoints go in OpenAPI **and** [api.md](./api.md).
6. Get a review, address comments, merge.

Keep PRs small and single-purpose.

## Local debugging

### Backend

- Run a single test: `cd backend && pytest tests/path/to/test_x.py::test_name -q`.
- Run uvicorn with reload: `uvicorn src.main:app --reload` and set `LOG_LEVEL=DEBUG` in `.env`.
- Inspect Celery tasks: `make worker` with `-l debug` (task bodies are stubs today).
- The Cognee seam is mockable — unit-test services with `memory_service` mocked. The SDK lives only in
  `backend/src/memory/cognee_client.py`.

### Frontend

- `pnpm dev` with React Query Devtools for server-state inspection.
- Network/API issues: check `NEXT_PUBLIC_API_BASE_URL` and CORS (`BACKEND_CORS_ORIGINS`).

### Database

- `make migrate` to apply migrations; `make migration m="…"` to create one. See
  [database.md](./database.md).
